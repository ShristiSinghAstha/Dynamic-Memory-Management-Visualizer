Memory Management Concepts

Paging

Memory is divided into fixed-size pages. Each page maps to a frame in physical memory.

  • Fixed size blocks
  • Avoids external fragmentation
  • Page Table for logical-to-physical mapping

Segmentation

Memory is divided by logical units like code, stack, and heap. Each has a base and limit.

  • Variable-sized segments
  • Better program structure visibility
  • Can lead to external fragmentation

Allocation Strategies

Explore how First Fit, Best Fit, and Worst Fit allocate memory to processes.

  • First Fit: fills first available hole
  • Best Fit: fills smallest suitable hole
  • Worst Fit: fills largest hole

FIFO (First-In-First-Out)

Pages are removed in the order they were loaded into memory, regardless of usage.

  • Oldest page is replaced first
  • Simple and queue-based logic
  • Can lead to suboptimal replacements

LRU (Least Recently Used)

Replaces the page that hasn't been accessed for the longest time.

  • More accurate than FIFO
  • Tracks access history
  • Can be expensive to implement efficiently
